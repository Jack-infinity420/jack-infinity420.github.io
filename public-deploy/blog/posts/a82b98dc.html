双锋猎鹰——低算力边缘平台的增强协同与轻量自主弹性目标检测系统

发表于2026-06-18|更新于2026-06-18|techAI
|总字数:2.5k|阅读时长:7分钟

项目概述

双锋猎鹰（Double Falcon） 是 2026 年中国高校计算机大赛—网络技术挑战赛的创意系列作品。项目面向低算力无人机边缘平台，以结构感知模型切割聚合双机算力，以 CAST V2 适应动态无线链路，并通过 LoRa 心跳与轻量模型接管保障任务持续在线。

MISSION SYSTEM ONLINE · LINK ADAPTIVE · MODE CO-INFERENCE

三大核心能力——双机协同分载、链路自适应、异常可接管——共同构成了一条完整的弹性目标检测流水线。

背景与动机

低空任务，正在从”空中采集”走向”机载智能”。

在电力巡检、应急救援、边境巡护、交通监测、灾害评估与目标搜索等场景中，无人机正从单纯的数据采集平台，演进为集感知、计算、通信与任务响应于一体的空中智能节点。然而，传统”空中采集—数据回传—地面处理”模式依赖持续、稳定的高带宽链路；而单机机载推理又受到功耗、散热、内存与算力限制。

“双锋猎鹰”因此提出”增强协同 + 轻量自主“的弹性感知路径，具体包括四个方向：

  1. 协同增强：通过结构感知切割，将 YOLO26m 分为前后端两部分，在两台边缘设备之间协同运行，突破单机算力瓶颈。
  2. 链路自适应：细腰化模型 + CAST V2 策略，使系统能够根据链路质量动态调整传输特征的数据量与质量。
  3. 弹性接管：借助 LoRa 独立心跳监测与轻量 YOLO26n，在协同失效时自动切换到单机检测，保证持续输出。
  4. 结果可回传：检测结果、运行状态与异常信息可回传至地面站，为任务决策与可视化指挥提供支撑。

核心挑战

低空边缘智能，真正难在多重约束同时存在。算力不足、特征体量、网络结构依赖与链路波动并非彼此独立：优化其中一项，往往会把压力转移到另一项。系统必须在精度、时延、通信与任务连续性之间进行整体权衡。

挑战具体问题
算力受限Jetson Nano 等低功耗平台难以直接稳定承载较大检测模型，实时性与模型规模天然冲突。
特征传输过大中间特征并不天然”轻”，层 10 的 P3/P4/P5 FP16 特征合计约 8.20 MB，远高于同尺寸 JPEG 图像。
结构依赖强YOLO Neck 对多尺度特征依赖敏感，通信压缩不能简单粗暴地”扔掉某个尺度”，否则会破坏后端输入结构。
任务不能中断低空链路会随距离、姿态和环境发生波动，因此系统必须具备异常检测、弹性降级与自主接管能力。

技术架构

协同检测流水线

双锋猎鹰以结构感知切割聚合两台边缘节点的算力，以 CAST V2 适应动态链路，再由 LoRa 心跳与 YOLO26n 完成异常接管，构成”协同增强—弱链路降级—异常接管—稳定恢复”的完整闭环。

切割边界：Layer 10（Backbone—Neck 自然边界），候选点均衡度达 0.95。系统通过 WiFi / 4G / 5G 链路将检测结果回传至综合地面站。

核心创新点

项目围绕四项核心创新，构成一条完整证据链：先选择结构上正确的切割边界，再从模型结构层面压缩传输规模，随后让特征质量随链路变化，最后以独立心跳与轻量模型保障任务连续性。

01 结构感知切割点选择算法（STRUCTURE-AWARE SPLIT）

解析跨层连接与保存特征，将结构完整性设为硬约束，再综合计算均衡、通信开销、工程复杂度与流水线效率进行评分。在 12 个 YOLO 变体中识别 276 个候选点，最终推荐 YOLO26m 在层 10 后切分。

  • 均衡评分：94.98
  • 结构评分：100
  • 综合得分：77.58

02 细腰式原生窄口（SLIM-WAIST MODEL）

在训练阶段同步考虑协同传输，将层 8、9、10 的通道由 512 缩窄至 256，形成紧凑的原生边界接口，无需额外引入 Encoder / Decoder。

  • 参数量：19.31M
  • mAP@0.5：0.9243

03 LoRa 异常检测与弹性接管（INDEPENDENT HEARTBEAT）

高速数据链路负责特征传输，低速 LoRa 链路专职状态核验。协同链路异常后自动切换至 YOLO26n 单机检测，链路稳定恢复后再回到增强协同。

  • 心跳频率：1Hz
  • 判定策略：连续 3 次心跳异常触发切换

04 CAST V2：降质量，不破坏结构（FULL-SCALE ALWAYS PRESENT）

由 CAST V1 的”按尺度丢弃”迭代为 V2 的”三尺度恒在场”。链路变差时只降低数值精度与空间细节，不直接移除 P3 / P4 / P5，使后端 Neck 始终获得结构一致的输入。

CAST V2：链路感知的传输编排

CAST V2 不是简单压缩，而是链路感知的传输编排策略。它将特征结构完整性作为前提，在不同带宽状态下动态选择不同传输档位，让协同推理在弱链路中依然可用。

在项目阶段性复测中，C1—C4 均实现 1/1 目标匹配；即使在最弱的 C4 档位下，也能保持 0.7332 的可检测保底置信度。有效载荷从 8200.05 KB 降至 280.74 KB（29.2× 压缩比），既保留多尺度结构，又显著降低链路压力。

实验验证

项目用阶段性实验，验证了”切得对、传得动、断了不断”三个核心命题：

  • 结构感知算法：验证切分位置与前后端一致性（Layer 10 均衡度 0.95，结构评分 100）
  • 细腰模型和 CAST V2：验证传输负载可控（C1→C4 压缩 29.2×），细腰 YOLO26m 保持 mAP@0.5 = 0.9243
  • LoRa 心跳与 YOLO26n：验证协同异常时仍可保持基本检测输出（1Hz 心跳 + 连续 3 次异常判定策略）

说明：页面中的数据与表述基于作品文档中的阶段性实验记录进行整理展示。真实外场长期链路稳定性、多目标复杂场景与完整无人机闭环仍属于后续补充验证范围。

原型验证

项目已完成前后端模型封装、CAST 传输、LoRa 心跳、地面站可视化、Jetson 双机验证及机载任务载荷集成设计，使核心方法能够落到真实硬件与通信链路上：

  • 综合地面站 Web 系统：航线规划、任务控制与状态回传一体化展示
  • 桌面双机验证平台：WiFi/TCP + LoRa 心跳双链路原型验证
  • Prometheus 科研无人机：机载计算、通信与载荷集成基础平台
  • 机载任务载荷集成底板：通信、电源与控制组件的一体化机载集成
  • 双光光电吊舱：面向目标感知的任务载荷
  • HOMER 无线图数传模块：面向双机协同与任务数据回传的高速通信链路

应用场景

从坦克样本验证，延展到更广阔的低空任务。虽然本项目以坦克样本作为阶段性验证对象，但核心方法并不依赖单一目标类别。更换数据与模型后，即可扩展到多类真实任务：

  • 电力与基础设施巡检：在算力与链路都受限的情况下，持续识别缺陷、异物与结构异常，提升远距离巡检的在线智能化水平。
  • 应急救援与灾害评估：在弱通信环境中快速发现受困人员、车辆、火点与关键区域，保证短时失联时依旧有基本感知输出。
  • 区域安防与边境巡护：面向复杂地形、远距离与动态链路条件，对人员、车辆与装备进行持续搜索、识别与上报。
  • 交通监测：识别拥堵、事故车辆与异常行为，通过地面站完成结果可视化、任务调度与辅助决策。
  • 森林防火与生态监测：对烟火、非法活动与生态变化目标开展边缘侧实时检测，并通过低带宽链路回传关键结果。
  • 低空智能集群：为多节点算力聚合、模型分载部署、链路自适应传输与异常弹性接管提供可迁移的系统方法。

结语

“双锋猎鹰”把目标检测模型、边缘计算资源、无线通信链路与无人机任务系统协同融合，形成一套更贴近真实低空应用场景的智能感知方案。它的价值不只在于能飞、能跑、能测，更在于面对约束与波动时，仍然能够稳定地持续工作。
