## 二、色彩系统问题（优先级：高）

### 2.1 色彩体系不统一

**问题**：`source/css/weather.css` 中大量使用硬编码颜色，未与 Butterfly 主题 CSS 变量融合。

```css
/* weather.css:15-29 */
:root {
  --rain-r: 150; --rain-g: 190; --rain-b: 225;  /* 浅蓝，偏冷 */
  --drop-r: 195; --drop-g: 218; --drop-b: 240;  /* 更浅 */
  --snow-opacity: 0.7;
  --sun-opacity: 0.12;
  --glass-alpha: 0.04;
}

/* weather.css:34-46 */
.weather-card {
  background: var(--card-bg, rgba(255,255,255,0.015));  /*  fallback 值过于透明 */
  border: 1px solid var(--card-border, rgba(255,255,255,0.04));
  border-radius: 14px;
  font-family: 'LXGW WenKai', ...;  /* 字体栈正确 */
}
```

**为什么不好**：
- 硬编码颜色 `#c8d2e0`、`#8899aa`、`#556677` 在 `weather.css:62-91` 中直接写死
- 浅色主题下卡片几乎不可见（`rgba(255,255,255,0.015)` 背景）
- 深色模式覆盖 `[data-theme='dark']` 在 `weather.css:475-507` 中过于粗暴，使用 `!important`

**建议**：建立完整的色彩变量体系，统一使用 CSS 自定义属性。

```css
/* 建议统一色彩变量 */
:root {
  /* 宇宙深邃色系 */
  --cosmos-deep: #0a0e1a;
  --cosmos-void: #0d1117;
  --cosmos-nebula: #1a1f3a;
  
  /* 古典诗意色系 */
  --poem-ink: #2c2c2c;
  --poem-wash: #f5f0e8;
  --poem-gold: #c9a96e;
  --poem-cinnabar: #b54a3f;
  
  /* 科技荧光色系 */
  --tech-cyan: #00f0ff;
  --tech-violet: #b829dd;
  --tech-matrix: #00ff41;
}
```

### 2.2 星空背景与卡片对比度不足

**问题**：截图显示侧边栏卡片在星空背景下文字颜色偏淡（`#c8d2e0` 在复杂背景上可读性差）。

```css
/* weather.css:270-278 */
.wc-welcome-greet {
  color: var(--text, #c8d2e0);  /* 浅灰蓝，在星空背景下对比度不足 */
  font-size: 16px;
}

.wc-welcome-prose {
  color: var(--text2, #8899aa);  /* 更浅，几乎融入背景 */
  font-size: 12px;
}
```

**为什么不好**：
- 星空背景有亮有暗，固定浅灰色在某些区域会消失
- 缺少文字阴影或背景遮罩保护可读性

**建议**：
```css
.wc-welcome-greet {
  color: rgba(255, 255, 255, 0.95);
  text-shadow: 0 1px 3px rgba(0, 0, 0, 0.5);  /* 阴影保护可读性 */
}
```

## 三、字体与排版问题（优先级：高）

### 3.1 字号层级跳跃过大

**问题**：侧边栏卡片内字号从 16px 到 8px，缺乏中间层级。

```css
/* weather.css 中的字号分布 */
.wc-welcome-greet { font-size: 16px; }   /* 问候语 */
.wc-fixed { font-size: 14px; }           /* 固定诗句 */
.wc-grav-planet { font-size: 14px; }     /* 行星名 */
.wc-greeting { font-size: 13px; }        /* 问候 */
.wc-poem { font-size: 13px; }            /* 诗词 */
.wc-weather { font-size: 13px; }         /* 天气 */
.wc-welcome-prose { font-size: 12px; }   /* 欢迎正文 */
.wc-location { font-size: 10px; }        /* 诗人名 */
.wc-grav-hint { font-size: 10px; }       /* 重力提示 */
.wc-mode { font-size: 8px; }             /* 模式按钮 ← 过小 */
```

**为什么不好**：
- 16px → 14px → 13px → 12px → 10px → 8px 的递减过于细碎
- 8px 在桌面端几乎不可读，违反 WCAG 可访问性标准
- 没有形成"标题-正文-注释"的清晰三级结构

**建议**：使用更克制的字号阶梯
```css
/* 建议三级字号体系 */
.wc-title { font-size: 18px; }      /* 主标题：问候语 */
.wc-body { font-size: 13px; }       /* 正文：诗词、信息 */
.wc-caption { font-size: 11px; }    /* 注释：诗人、元信息 */
```

---

### 3.2 缺少书法字体层次

**问题**：当前使用 `'LXGW WenKai'` 作为主力字体，但缺少衬线体搭配。

```css
/* weather.css:45 */
font-family: 'LXGW WenKai', 'Noto Sans SC', 'PingFang SC', 'Microsoft YaHei', serif;

/* weather.css:63 */
font-family: 'Cormorant Garamond', 'LXGW WenKai', 'Noto Serif SC', serif;
```

**为什么不好**：
- `Cormorant Garamond` 是西文衬线体，对中文无效果
- 缺少真正的中文书法/刻本字体（如 `方正清刻本悦宋`、`汉仪尚巍手书`）
- 古典诗意需要"墨韵"，当前字体过于现代

**建议**：引入分层字体栈
```css
/* 标题层：书法感 */
font-family: 'LXGW WenKai', '方正清刻本悦宋', 'Source Han Serif SC', serif;

/* 正文层：刻本感 */
font-family: 'LXGW WenKai', 'Noto Serif SC', 'PingFang SC', serif;

/* 科技层：等宽 + 无衬线 */
font-family: 'SF Mono', 'Noto Sans SC', monospace;
```

## 四、空间与留白问题（优先级：中）

### 4.1 间距数值随意

**问题**：margin/padding 数值缺乏规律。

```css
/* weather.css */
.weather-card { padding: 18px 15px 14px; }   /* 不对称 */
.wc-welcome { margin-bottom: 18px; padding-bottom: 16px; }
.wc-fixed { margin-bottom: 10px; }
.wc-greeting { margin-bottom: 4px; }
.wc-poem { margin-bottom: 1px; }              /* 1px 无意义 */
.wc-location { margin-bottom: 12px; }
```

**为什么不好**：
- `1px` 间距在视觉上不产生任何效果
- 18px / 16px / 15px / 14px 过于接近，无法形成节奏感
- 建议基于 4px 或 8px 网格系统

**建议**：使用 8px 基准网格
```css
.weather-card { padding: 16px; }
.wc-welcome { margin-bottom: 16px; padding-bottom: 16px; }
.wc-section { margin-bottom: 12px; }   /* 1.5 × 8px */
.wc-tight { margin-bottom: 8px; }      /* 1 × 8px */
```

### 4.2 卡片圆角不统一

**问题**：卡片内部元素圆角与卡片本身不一致。

```css
.weather-card { border-radius: 14px; }     /* 卡片 */

.wc-grav-qbtn { border-radius: 10px; }     /* 重力按钮 */

**为什么不好**：圆角数值过于接近（14px / 12px / 10px），视觉上混乱。

.card { border-radius: 16px; }      /* 外容器：大圆角 */
.button { border-radius: 8px; }     /* 按钮：中等 */
.pill { border-radius: 100px; }     /* 标签：完全圆角 */
```

## 五、动态与交互问题（优先级：高）

### 5.1 卡片缺少悬浮效果

**问题**：与 blog.liushen.fun 对比，我们的卡片 hover 时没有阴影加深或位移效果。

```css
/* 当前 weather.css:34-46 —— 无 hover 效果 */
.weather-card {
  background: var(--card-bg, rgba(255,255,255,0.015));
  border: 1px solid var(--card-border, rgba(255,255,255,0.04));
  border-radius: 14px;
  transition: background 0.5s, border-color 0.5s;  /* 仅背景过渡 */
}

/* blog.liushen.fun 的做法 —— 悬浮加深阴影 */
.card-widget:hover {
  box-shadow: var(--card-hover-box-shadow);  /* 0 3px 8px 6px rgba(7,17,27,0.2) */
}
```

**为什么不好**：卡片是静态的，缺乏"生命力"，不符合未来科技的动态美学。

**建议**：
```css
.weather-card {
  transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);
}

.weather-card:hover {
  transform: translateY(-2px);
  box-shadow: 0 8px 32px rgba(0, 240, 255, 0.15);  /* 青色科技光晕 */
}
```

---

### 5.2 按钮过渡生硬

**问题**：按钮 hover 仅改变边框颜色，过于简单。

```css
/* weather.css:108-133 */
.wc-mode {
  transition: all 0.3s;  /* 无缓动函数 */
}

.wc-mode:hover {
  border-color: var(--accent, #6e96c4);
  color: var(--accent, #6e96c4);
}
```

**为什么不好**：`transition: all 0.3s` 没有指定缓动函数，默认是 `ease`，效果平庸。

**建议**：
```css
.wc-mode {
  transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);
  position: relative;
  overflow: hidden;
}

.wc-mode::before {
  content: '';
  position: absolute;
  inset: 0;
  background: var(--accent);
  opacity: 0;
  transition: opacity 0.3s;
}

.wc-mode:hover::before {
  opacity: 0.1;  /* 微光填充 */
}
```

---

### 5.3 天气 Canvas 渲染异常

**问题**：之前出现过彩虹同心圆环的渲染 bug，canvas z-index 设置导致遮挡问题。

```css
/* weather.css:2-10 */
#weather-particles-canvas {
  position: fixed;
  top: 0; left: 0;
  width: 100%; height: 100%;
  z-index: -1;           /* 在最底层，可能被某些元素覆盖 */
  pointer-events: none;
}
```

**为什么不好**：
- `z-index: -1` 在某些浏览器/主题组合下可能导致渲染异常
- 没有 `mix-blend-mode` 或其他融合效果，粒子与背景生硬叠加

**建议**：
```css
#weather-particles-canvas {
  position: fixed;
  top: 0; left: 0;
  width: 100%; height: 100%;
  z-index: 0;            /* 改为 0，配合 isolation */
  pointer-events: none;
  mix-blend-mode: screen; /* 粒子与星空背景融合 */
}
```

## 九、改进优先级路线图

### 第一阶段：基础修复（1-2 天）
1. **统一色彩变量** — 建立完整的 CSS 变量体系

3. **水墨晕染背景** — 卡片内淡金色/墨色渐变

### 第三阶段：宇宙氛围（3-5 天）

3. **粒子系统优化** — 与背景融合，修复 z-index

### 第四阶段：科技元素（3-5 天）
